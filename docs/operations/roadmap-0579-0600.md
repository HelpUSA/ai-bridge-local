# Roadmap 0.5.79 to 0.6.0

## Decision

AI Bridge Local becomes the durable control plane. The extension becomes a thin browser adapter.

## 0.5.79 - Boundary, invariants and stale PID hotfix

- Document official control plane boundary.
- Add control plane invariants.
- Add conservative stale PID hotfix: remove PID file only when PID does not exist.
- Draft Command Contract v2.
- Introduce trace_id or correlation_id in new records where possible.

Acceptance:

- Docs state that the extension never owns workflow decisions.
- Simulated stale temp/brain_worker.pid can be recovered without manual deletion.
- No aggressive process kill is introduced.

## 0.5.80 - BrowserEvent, BrowserAction and QueueAdapter minimum

- Persist BrowserEvent v1.
- Persist BrowserAction v1.
- Require action_result for browser actions.
- Add minimum QueueAdapter interface: enqueue, claim, ack, fail and heartbeat.

Acceptance:

- Opening a chat can create a snapshot or event.
- A requested inject action records sent, failed, unavailable, expired or deduped.
- Queue code starts using the adapter boundary.

## 0.5.81 - Worker supervisor

- Add worker_instance_id.
- Add heartbeat for idle and running workers.
- Add stale PID recovery with PID and start_time validation.
- Add Bridge Doctor.
- Add safe worker restart policy.

Acceptance:

- Stale PID is recovered automatically.
- Worker liveness is visible without manual log inspection.

## 0.5.82 - Durable Queue v2

- Add commands_v2, runs and dead_letters.
- Add lease, ack, retry, timeout and dedupe.
- Persist security_decision and risk_level.

Acceptance:

- Commands do not stay indefinitely queued without diagnosis.
- Lease expiration leads to retry, needs attention or dead letter.

## 0.5.83 - Command Contract v2

- Add compact command format.
- Add payload_ref.
- Add source_chat_id inference.
- Add target aliases.
- Add structured validation errors.
- Add payload size limits.

Acceptance:

- A chat can trigger a recipe with a short intent message.
- Long scripts live locally, not inside chat envelopes.

## 0.5.84 - Recipes MVP

- safe_status.
- git_clean_guard.
- safe_patch.
- safe_patch_test.
- safe_patch_test_commit.
- inspect_failure.
- recover_stale_worker.

Acceptance:

- Patch, test and commit flow can run by recipe and return a concise summary.

## 0.5.85 - Result summarizer

- Add AI_LOCAL_RUN_SUMMARY.
- Store full logs by run_id.
- Return concise summary by default.
- Include error_class, root_cause and next_action.
- Apply redaction before chat delivery.

Acceptance:

- Failures produce short summaries sufficient to choose the next action.

## 0.5.86 - Chat Watchdog MVP

- Start with observe_only mode.
- Later add active_resume.
- Add cooldown, dedupe_key and max_resumes_per_task.
- Detect idle chat, queued too long, final result without continuation and UI error.

Acceptance:

- Watchdog records needs_attention without message loops.
- Active resume is bounded and deduped.

## 0.5.87 - Safe large command payload transport

- Keep compact command arguments inline while their UTF-8 JSON representation is at most 32768 bytes.
- Store larger argument objects through `POST /v1/payloads`.
- Submit the compact command with empty inline `args` and the returned SHA-256 `payload_ref`.
- Resolve and merge the durable JSON payload inside the existing command plane before capability execution.
- Preserve the legacy gateway on port `8766`.
- Preserve the compact command plane on port `8767`.
- Do not add payload resolution to the legacy queue or Brain Worker.

Acceptance:

- Small commands preserve the existing inline path.
- Large commands use one durable payload reference.
- The payload reference matches `sha256:<64 lowercase hexadecimal characters>`.
- Existing command-plane payload storage and execution contracts remain valid.
- Browser, Python and documentation regression tests pass.

## Historical superseded proposal - Control Center v2

The former `0.5.87 - Control Center v2` proposal was created on
2026-07-06 and was superseded by the authoritative M12 definition
published with the M11 completion checkpoint on 2026-07-20.

Preserved historical proposal:

- Add chat panel.
- Add queue panel.
- Add worker panel.
- Add runs panel.
- Add dead letters panel.
- Add browser actions panel.
- Add retry, cancel, resume and inspect operations.

Historical acceptance:

- The user can identify why work stopped without opening DB or logs manually.
## 0.5.88 - Observability

- End-to-end trace_id.
- Queue metrics.
- Time per state.
- Retry counts.
- Worker uptime.
- Structured event log.

Acceptance:

- Every command has a visible timeline from chat to result delivery.

## 0.6.0 - Control plane baseline

0.6.0 is reached when thin extension mode, durable queue, worker supervisor, command contract v2, recipes, summarizer and observe-only watchdog are stable with legacy compatibility.
