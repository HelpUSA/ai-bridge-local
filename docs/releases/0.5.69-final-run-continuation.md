# 0.5.69 final run continuation

## Summary

0.5.69 fixes the local runner continuation contract so final AI_LOCAL_RUN messages can wake the chat instead of leaving it stuck after a queued notice.

## Problem

The worker emitted final AI_LOCAL_RUN results with result_is_final=1 and chat_can_continue=1, but still used no_reply=1. That combination was contradictory: the result said the chat could continue, but also told it not to reply.

## Fix

- Added final_no_reply in brain_worker.py.
- Final AI_LOCAL_RUN now emits no_reply=0 when chat_can_continue=1.
- Added smoke_final_run_continue_no_reply_0569.py.
- Added smoke_post_command_duplicate_idempotent_0569.py.

## Validation

- py_compile passed for brain_worker.py and the new continuation smoke.
- node --check passed for extension/background.js, extension/content_script.js and extension/route_classifier.js.
- smoke_final_run_continue_no_reply_0569 passed.
- smoke_post_command_duplicate_idempotent_0569 passed.
- smoke_direct_interchat_router_0542 passed.
- smoke_gateway_feedback_dedup passed.
- smoke_command_accepted_progress_notice passed.

## Operational note

The AI_LOCAL_RUN generated by the command that patches the worker may still be formatted by the old worker process. Restart or reload the local worker and extension before validating the new no_reply=0 final-result behavior in live runs.
