# 0.5.70 gateway final fallback results

## Summary

0.5.70 completes the continuation fix by updating gateway_local.py. The gateway fallback path can emit final AI_LOCAL_RUN messages, so it must also wake the chat when chat_can_continue=1.

## Problem

0.5.69 fixed brain_worker.py, but live testing showed final results could still come back with no_reply=1. Inspection found another final-result path in gateway_local.py.

## Fix

- Updated gateway_local.py final fallback AI_LOCAL_RUN from no_reply=1 to no_reply=0.
- Preserved queued gateway feedback as no_reply=1.
- Bumped visible local bridge version to 0.5.70.
- Added smoke_gateway_final_run_no_reply_0570.py.
- Added smoke_post_command_duplicate_idempotent_0570.py.

## Validation

- Live run returned no_reply=0 with result_is_final=1 and chat_can_continue=1.
- py_compile passed for gateway_local.py, brain_worker.py and the new smokes.
- node --check passed for extension/background.js, extension/content_script.js and extension/route_classifier.js.
- smoke_gateway_final_run_no_reply_0570 passed.
- smoke_final_run_continue_no_reply_0569 passed.
- smoke_post_command_duplicate_idempotent_0570 passed.
