# 0.5.71 refresh direct target before delivery

## Summary

0.5.71 reduces false direct-delivery acknowledgements by refreshing the target ChatGPT tab before sending an inter-chat message.

## Problem

Some result_to messages were marked acked by the database and the extension returned button_click_confirmed or enter_confirmed, but the message was not visible in the current chat. Inspection showed the direct-delivery path could reuse registry[targetChatId] before rediscovering the current tab.

## Fix

- The direct-delivery path now rediscovers the target tab before sending.
- Target discovery now prefers an active tab that matches the target chat id or target_url.
- Version markers were bumped to 0.5.71.
- Added smoke_refresh_direct_target_0571.py.

## Validation

- post_reload_live_smoke_0571_20260701_001 passed with VERSION 0.5.71 and final no_reply=0.
- interchat_live_0571_target_url_20260701_003 returned sent_direct with versao=0.5.71.
- smoke_refresh_direct_target_0571 passed.
- smoke_gateway_final_run_no_reply_0570 passed.
- node --check passed for extension/background.js, extension/content_script.js and extension/route_classifier.js.

## Remaining issue

The parser is still sensitive to raw line breaks inside JSON string fields such as message or from_agent. This is a candidate for 0.5.72 parser hardening.
